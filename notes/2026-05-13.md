## NOTES Version of Git and Python + Initial Commit

- To check Python version use python --version or python3 --version -> Python 3.12.4
- To check git version use git --version -> git version 2.51.0
- To cinfigurate git use:
    git config --global user.name "Your Name"
    git config --global user.email "your-email@example.com"
- To check the version use:
    git config --global --list
- To do the first commit:
    git init
    git status
    git add .
    git commit -m "Initial repo structure"
- To push the project to GitHub:
    git branch -M main
    git remote add origin YOUR_REPOSITORY_LINK
    git push -u origin main

## Mini-test

1. What is the difference between Git and GitHub?

Answer:
Git is used locally and GitHub is saved on servers.

Better version:

Git is a version control tool used mainly locally to track changes in a project. GitHub is an online platform where Git repositories can be stored, shared, and collaborated on.

2. What is the difference between commit and push?

Answer:
Commit saves the changes to our local databse on git and push "moves" it to the server.

Better version:

A commit saves staged changes in the local Git repository. A push uploads local commits to the remote repository, for example GitHub.

3. What is README.md used for?

Answer:
README.md is used to explain how our code works, what it is used for, how should we install it. For a new user to have a easy start.

Better version:

README.md explains what the project is, what it is used for, how to install or run it, and helps a new user understand the project quickly.

4. Why should I keep my learning work on GitHub?

Answer:
If something happens with computer, data (code and changes) is saved on servers.

Better full answer:

I should keep my learning work on GitHub because it backs up my code, tracks my progress, shows my consistency, and can later become part of my portfolio for recruiters.

5. What does git status show?

Answer:
Git status shows if our changes made to the code / files are commited and saved (pushed).

Better version:

git status shows the current state of the repository: changed files, untracked files, staged files, and whether there is anything to commit. It can also show if the local branch is ahead of or behind the remote branch.

6. What does git add . do?

Answer:
Git add . adds all files to the next commit. If we enter file name instead of the . we will get only that certain file commited in the next commit. 

Better version:

git add . stages all changed and new files in the current folder for the next commit. If I use a specific file name, only that file is staged.

7. What does git commit -m "message" do?

Answer:
It commits the files that were added to this commit to local database and saves it under the name "message", so that we can easily find the commit we need, and now what we did in this commit. 

Better version:

git commit -m "message" saves the staged changes into the local Git history with a message describing what changed. Each commit also receives a unique ID.

8. What does git push do?

Answer:
Push sends the changes from the local git to the server on GitHub 

Better version:

git push uploads local commits from my Git repository to the remote repository on GitHub.
